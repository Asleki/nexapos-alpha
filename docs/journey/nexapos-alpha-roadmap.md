# Engineering Principles

- Event-first architecture
- Offline-first execution
- Immutable event history
- Read-model driven UI
- Privacy-first identity
- One immutable Nexa ID per verified citizen
- Business modules own operational data only
- Identity remains centrally managed
- Simulation and production remain permanently separated
- All external integrations must use adapter contracts
- Future APIs are prepared during Alpha but are not implemented until NexaPOS Alpha 1.0 is stable
